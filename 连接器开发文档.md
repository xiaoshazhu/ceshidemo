> **版本**: V3.2.0

## 一、 飞书连接器总体界面与操作流

> [!IMPORTANT]
> **弹窗唤起与配置时序说明：**
> 本连接器是完全嵌入在飞书多维表格环境中的插件，无独立系统后台。
> 1. **弹窗唤起效果**：用户在多维表格的“数据同步”或“自动化”节点中点击配置时，首先在多维表格页面中以**弹窗（Modal Dialog）**形式打开“平台数据源连接控制台”主界面（即**“主配置弹窗”**）。
> 2. **在弹窗内配置**：在此主配置弹窗打开后，用户方可依次进行数据源选择、账号授权、参数和字段配置。
> 3. **保存返回**：所有配置完成后，点击【创建/保存】才可将任务配置回传给多维表格引擎，随后关闭并销毁弹窗，开始后续的定时同步。

本连接器采用**左侧导航快捷菜单（锚点）、右侧单页长滚动配置表单、底部常驻操作按钮**的整体结构。

```
[用户在多维表格中触发] 
          │
          ▼
┌──────────────────────────────────────────────────────────────────────────────┐
│ ◀ 抖店                                                                      X │
├──────────────┬───────────────────────────────────────────────────────────────┤
│ [🔗 数据源选择]│ 📍 1. 数据源选择区块 (单选，切换平台时下方所有区块动态更新)     │
│              ├───────────────────────────────────────────────────────────────┤
│ 🔑 账号设置   │ 📍 2. 账号设置区块 (展示已连卡片，支持拉起切换账号弹窗)          │

│ ⚙️ 参数设置   │ 📍 3. 参数设置区块 (支持红星*必填、ⓘ提示、并排下拉框、折叠选项)   │
│              ├───────────────────────────────────────────────────────────────┤
│ 📊 字段设置   │ 📍 4. 字段设置区块 (配置多维表格子表并支持可视化连线映射)       │

└──────────────┴───────────────────────────────────────────────────────────────┘
```

## 二、 侧边栏整体框架布局与底部动作按钮 (Main Window Setup)

配置窗口整体高度自适应，底部常驻有【取消】与【创建】（或编辑状态下的【保存】）两个操作按钮。

### 2.0.1 整体布局与滚动跳转行为

1. **左侧锚点导航 (Left Anchor Navigation)**：
   * 左侧菜单宽度固定（例如 `160px`），项目包含：`数据源选择`、`账号设置`、`参数设置`、`字段设置`、`同步设置`，并且在左下角固定放置带问号图标的 `使用说明`。
   * 选中项（如 `账号设置`）呈现浅紫色/淡蓝色高亮背景，文字加粗，辅以专属的图标。
   * 点击左侧菜单项时，触发 `JavaScript` 平滑滚动行为，使右侧对应区块顶部滚动对齐容器顶部：
     ```javascript
     // 示例滚动实现
     const sectionEl = document.getElementById('section-account-settings');
     sectionEl.scrollIntoView({ behavior: 'smooth', block: 'start' });
     ```
2. **右侧长滚动容器 (Right Scrollable Container)**：
   * 右侧为一整条连续滚动的表单区域（CSS 属性 `overflow-y: auto; scroll-behavior: smooth;`）。
   * 包含 5 个区块，每个区块用一个独立的 `<section>` 或 `<div id="section-*">` 包裹。
3. **滚动监听与高亮 (Scrollspy)**：
   * 在右侧容器滚动时，前台通过监听 `scroll` 事件或使用 `IntersectionObserver` 监控当前位于可视区域（Viewport）顶部的区块，并动态高亮左侧侧边栏中对应的菜单项。
4. **选择平台后，下方自动更新内容 (Dynamic Platform Rendering)**：
   * 当用户在第 1 区块（数据源选择）改变平台单选值时，右侧容器中的 2（账号设置）、3（参数设置）、4（字段设置）、5（同步设置）等所有区块内容及左下角 `使用说明` 会**瞬间刷新并局部更新渲染**为该平台的特定模板，此时已选账号或已映射表单若发生冲突则予以清空，但不影响页面其他部分的状态。

### 2.0.2 底部常驻按钮功能表

| 按钮名称 | 组件样式 | 触发场景 | 交互行为与后端动作 |
| :--- | :--- | :---: | :--- |
| **【取消】** | `el-button` / 灰底黑字、浅灰色边框，置于右下角 | 用户希望放弃当前配置并关闭弹窗 | 销毁当前 H5 状态，直接关闭飞书配置弹窗。 |
| **【创建】/【保存】**| `el-button` / 蓝底白字，置于右下角 | 用户完成全部必填配置，字段映射成功后激活（默认为置灰状态） | 1. 提交整套配置 JSON 到后端。`POST /api/v1/sync/tasks/save`。<br>2. 后端保存数据表映射与平台拉取参数。<br>3. 成功后关闭弹窗，多维表格对应列开始打上数据连接角标。 |

#### 3.2.5 页面按钮清单 (Button Specifications)
*   **【已选账号卡片】**：点击拉起“切换账号”模态弹窗。
*   **【... 按钮】**：点击卡片右侧的三个点，同样能拉起“切换账号”模态弹窗。
*   **【关联其他账号】**：位于切换账号列表最底部，点击拉起“步骤 1：首屏来源类型选择”弹窗。
*   **【下一步】与【返回上一步】按钮**：用于在步骤 1 与步骤 2A/2B 页面之间切换，暂存用户在前一页的选择。
*   **【🔑 点击开始网页模拟登录】**（蓝底白字）：位于步骤 2B 模拟登录选项中。
    1. **弹窗层级**：点击后在当前**“主配置弹窗”**上方拉起一个独立的 WebView 内嵌窗口（**“网页登录子弹窗”**），专门用于显示第三方登录页面。
    2. **自动捕获与上报**：用户登录成功后，前端代理容器自动拦截捕获 Cookie/签名并调用后端 `POST /api/v1/connector/sources/login-capture` 进行验证与记录。
    3. **自动销毁机制**：一旦后端验证通过并返回成功，**该“网页登录子弹窗”必须立即自动销毁/关闭**，用户焦点无缝返回至仍在下方保持原状态的**“主配置弹窗”**。此时“网页模拟登录”按钮状态自动变更为 `[凭证已拦截]`，用户方可继续填写后续的参数设置和字段设置。
*   **【确认关联并绑定】按钮**：校验表单。若为步骤 2A 则调用挂载引用 API；若为步骤 2B 验证可用后关闭弹窗，将新账号追加至账号列表中。

#### 3.3.2 各平台特有的动态字段与开发参数模拟对照表
根据架构规范，**此处列出的所有平台在参数设置区块均完整支持模式 A (官方 API) 与模式 B (模拟登录网页捕获) 两种接入方式**：
* **在模式 A 下**：表单配置字段直接作为官方 OpenAPI 的入参传递。
* **在模式 B 下**：系统会优先调用三方平台后台的导航菜单接口获取路由树，再将配置 of 表单参数（如店铺ID、同步天数、过滤指标等）动态装配并推送到解析出的真实接口中进行抓取。

以下为各平台开发参数的详细对照：

## 四、 飞书连接器后端 Action API 交互规范

连接器 H5 配置页的交互动作及参数保存，统一通过连接器后台的标准数据动作（Action API）与多维表格引擎交互。

### 1. `get_auth_methods` (获取鉴权与平台列表)
*   **请求路径**：`GET /api/v1/connector/auth-methods`

### 2. `save_captured_credentials` (保存拦截凭证 - 模式 B)
*   **请求路径**：`POST /api/v1/connector/sources/login-capture`

### 3. `execute_sync_action` (数据获取并批量写入动作)
*   **请求路径**：`POST /api/v1/connector/sync/execute`
*   **核心逻辑**：后台在“每小时刷新”定时器触发或用户手动同步时执行。依据所选模式采取不同数据捕获链路：
    *   **模式 A (官方 API)**：依据映射配置和平台 API 参数直接请求官方 OpenAPI 接口，将返回的数据通过队列分批写入飞书数据表。
    *   **模式 B (模拟登录网页捕获 - 优先拉取控制台菜单结构)**：
        为规避三方平台改版导致的数据请求接口硬编码失效，以及妥善处理短信验证码/扫码等交互式登录在定时同步下的运行，后端必须采用以下动态发现与拉取流：
        1. **凭证复用与保活机制 (Credentials Reuse & Keep-Alive)**：
           * 后端执行定时同步时，**严禁再次唤起登录逻辑**，必须直接复用已在账号配置中拦截并加密存储的 Cookie 凭证及浏览器 UA 指纹。
           * **心跳延长**：后端服务应具备轻量级心跳模块，每 30 分钟模拟一次对目标平台的安全性公开接口（如店铺名/菜单查询接口）的请求，激活并保持会话状态，延长 Cookie 有效期。
           * **环境持久化**：内嵌浏览器沙箱容器（Puppeteer / Playwright）需配置固定的持久化目录（`userDataDir`），保留历史 LocalStorage 与 Session 缓存。在后续重新连接时，最大化提升一键登录（免验证码输入）的成功率。
        2. **拉取菜单树 (Fetch Menu Structure)**：利用拦截并处于保活状态的加密 Cookie 凭证，首先请求目标平台控制台的导航菜单树接口（例如：抖店罗盘 `/compass/api/v1/menu`，千帆后台 `/ark/api/v1/menu` 或千牛的路由树接口），获取页面布局和查询配置。
        3. **解析目标路由 (Resolve Endpoint)**：在返回的菜单树结构中，利用关键字或静态标识检索匹配当前同步模块（如“素材报表”或“经营概览”）对应的真实数据 Endpoint 路径与入参格式。
        4. **动态请求数据 (Fetch and Ingest Data)**：以解析出的 Endpoint 组装请求（结合时间区间、商家ID等），发起请求抓取数据，最终扁平化输出并写入多维表格。
        5. **凭证过期与重连闭环 (Retry / Callback Flow)**：若同步请求因 Cookie 彻底失效返回 401（未授权），同步任务静默退出。系统必须将错误类型（`凭证失效(Cookie过期)`）及其详情追加写入【异常对账监控多维表格】，自动触发飞书群聊警报。管理员在 H5 弹窗中点击【重新连接】以再次拉起网页登录子弹窗，输入验证码或扫码重新捕获凭证。

## 五、 全局异常监控与“异常多维表格”自动归档机制

为了彻底简化前端交互，本连接器**在前台配置弹窗中完全不提供任何运行日志、运行状态监控、预警频率或报警触发的 UI 配置项**。所有对于数据同步自动运行中的接口异常管理，均通过后端自动分发并静默写入到开发团队指定的**特定异常对账监控多维表格**中。

### 1. 自动写入特定异常监控多维表格与事件响应

当连接器后端执行定时同步（如每小时刷新）或用户手动点击同步时，若出现数据拉取或写入失败，处理流程如下：
1. **异常静默捕获**：若发生 `Cookie过期/失效 (模式B)`、`接口响应 500/401 (模式A)`、`接口入参校验失败`、或 `飞书多维表格 API QPS限频熔断` 等任意错误，后端同步核心将捕获错误异常，并中断当前同步事务。
2. **提取错误元数据**：后端从错误上下文及平台返回的 JSON 响应中提取结构化元数据（见下方定义）。
3. **写入特定的飞书多维表格**：后端读取内置配置的异常对账表 ID（`AppToken` & `TableID`），静默调用多维表格新增记录 API，将提取 of 错误信息作为一行新数据追加至该特定监控表中。
4. **触发飞书原生流转通知**：由于异常日志已写入多维表格，管理员或运营人员可直接在该表格中利用飞书原生的**“自动化助手”**（如：“当有新记录添加且处理状态为待处理时，向值班群发送飞书机器人消息”），无需连接器 H5 前端做任何定制开发或展示预警配置。

### 2. 异常对账监控多维表格 Schema 字段定义
开发人员需在后端异常写入模块对齐以下飞书多维表格结构，以便运营与技术支持人员直接在飞书内部排查处理：

| 表格字段名 | 字段类型 | 说明 | 示例值 |
| :--- | :--- | :--- | :--- |
| **异常任务ID (ID)** | 单行文本 | 唯一同步任务主键，用于定位出错的连接 | `TASK_20260615_01` |
| **异常发生时间** | 日期 (带时间) | 后端捕获到错误的具体服务器时间 | `2026-06-15 14:30:15` |
| **对接平台** | 单选 | 故障发生的第三方数据源平台 | `小红书蒲公英` / `抖音电商罗盘` |
| **账号/店铺名称** | 单行文本 | 授权失效或报错的账号名称/店铺名称 | `品牌运营主账号01` |
| **错误分类** | 单选 | 便于运营进行分类筛选和归属跟进 | `凭证失效(Cookie过期)` / `接口500报错` / `多维表格写入超限` / `网络请求超时` |
| **报错信息明细** | 多行文本 | 平台原始返回的错误 Response 文本或代码 StackTrace，方便定位问题 | `{"code": 401, "message": "session expired, please login again"}` |
| **处理状态** | 单选 | 便于在表格内协同跟进处理状态 | `待处理` / `已恢复(自动)` / `已修复(手动)` |
