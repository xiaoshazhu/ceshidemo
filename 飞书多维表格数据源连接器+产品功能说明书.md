# 飞书多维表格数据源连接器 - 产品功能说明书

本说明书站在业务协作者与运营人员的角度，对“多源电商与营销数据对接飞书多维表格（Bitable）”的业务场景进行功能设计与规划。本连接器采用嵌入在飞书多维表格中的**平台数据源连接控制台弹窗**形式，将阿里妈妈、抖音（抖店/千川/罗盘）、京麦、聚水潭、千牛工作台、小红书等平台的数据流打通，帮助业务人员在多维表格中直接完成自动化报表配置与同步。

---

## 一、 系统概述与设计目标

为解决运营与财务团队多平台手工导表效率低、数据滞后的问题，本连接器旨在打造一个**“配置即连接、无感自动同步”**的轻量化 H5 数据桥接弹窗。用户无需具备编程基础，只需在多维表格拉起的连接控制台弹窗中完成账号授权与字段映射，即可让飞书多维表格自动获取每日最新销售、广告投放、账单明细及 ERP 出入库数据。

---

## 二、 系统使用与连接模型

为降低业务协作门槛，本连接器**不设复杂的权限控制模型 (RBAC)**。连接器面向多维表格内的所有协作者完全开放，确保任何有权限编辑表格的用户均能自由连接、配置与同步数据。

### 1. 简易连接与配置权限
*   **全员可连接**：多维表格的任何协作者（如运营、财务、数据分析师）均可直接调用该连接器。
*   **免审批授权**：无需经过超级管理员审批，用户可根据自身业务需求，直接在多维表格的“数据同步”界面中选择并绑定对应平台的账号密钥或扫码授权。
*   **自定义配置**：每个协作者均可独立管理自己创建的数据源映射规则（Schema Mapping），根据具体表格列名进行拖拽映射。

### 2. 数据源与表单绑定模型
*   **一表对一源**：支持在一个多维表格中挂载多个连接任务。不同的业务人员可分别绑定不同的店铺数据或广告账号，并将数据定向同步至不同的数据表中。
*   **资产与配置共享**：
    *   **配置共享**：某一协作者创建的同步任务和字段映射配置，其他协作者均可查看并协同触发“立即同步”。
    *   **数据共享**：数据同步落地至多维表格后，自动成为多维表格的公共数据记录，所有拥有表格读写权限的人员均可基于该数据进行二次统计与 BI 报表设计。

### 3. H5 配置弹窗单页滚动与平滑锚点定位 (Anchor & Scrollspy Layout)
为了提供与飞书官方一致的、无缝且流畅的配置体验，弹窗采用“一屏到底 + 侧边锚点 + 底部常驻按钮”的现代设计：
*   **左侧锚点导航 (Left Sidebar Anchors)**：左侧常驻导航链接，项目包含：`数据源选择`、`账号设置`、`参数设置`、`字段设置`、`同步设置`，并且在左下角固定放置带问号图标的 `使用说明`。用户点击菜单时，右侧长滚动区域平滑滚动 (`Scrollspy Scroll`) 跳转定位到对应区块。选中菜单项时，呈现浅紫色/淡蓝色高亮背景，文字加粗，辅以对应的主题图标。
*   **右侧单页长滚动 (Right Long-Scrollable Form)**：所有配置区块排布于右侧长页面中，形成直观的连续表单。当用户在右侧上下滚动时，左侧菜单会根据当前可视区块自动高亮。
    *   **区块表单设计**：每个区块标题字号较大。输入项包含红色星号 `*` 必填标识、灰色圆圈信息提示图标 `ⓘ`；输入框为圆角矩形，包含浅灰色占位提示；时间范围等参数支持双下拉框横向并排；支持蓝色的“展开高级选项 ∨”折叠联动。
*   **第一步单选平台驱动渲染 (Platform Dynamics)**：用户在首个“数据源选择”区块切换单选平台（如“抖店”切换为“聚水潭 ERP”），右侧下方其余区块对应的账号授权类型、API 动态参数和帮助文档将在页面中实时更新渲染，无需重新加载整个弹窗。

## 三、 核心业务模块详情

### (一) 第三方平台授权与双模式凭证中心 (Credential & Login Hub)
作为跨平台数据同步的前提，负责各对接渠道的安全授权和生命周期管理。本模块在界面与账号绑定流程上做出了如下人性化与安全性优化：

*   **1. 全平台通用 A/B 双通道模型 (Universal Dual-Mode)**：
    *   为了给商户提供最灵活的接入能力，**连接器内所有对接的第三方平台在底层逻辑上均统一支持模式 A（官方 API 授权）与模式 B（网页模拟登录捕获）**。用户可根据自身是否具备开发者开放资质，自由切换绑定通道。
*   **2. 账号关联与切换卡片弹窗 (Account Switch & Card Binding Dialog)**：
    *   **已选账号卡片**：在右侧的“账号设置”区块中，已连接的账号以白色圆角卡片形式展示，包含橙色圆形头像（内有白字首字“王”）、账号/店铺名称（如“我的“抖店”连接器”），右侧有操作菜单触发点 `...` 按钮。
    *   **账号切换/关联弹窗**：当点击已选账号卡片或 `...` 菜单时，系统在页面中央弹出“切换账号”的独立模态对话框，包含以下层级和视觉元素：
        1. **提示信息**：顶部标题为 `切换账号` (带 `X` 关闭按钮)，下方有灰色提示文本 `请重新选择一个账号或关联其他账号`。
        2. **共享与个人账号卡片列表**：卡片行显示橙色“王”字圆形头像、账号别名，右侧在当前选中的账号行显示蓝色对勾 `✓` 标记。
        3. **关联其他账号**：列表最下方为添加新连接项，显示虚线圆形头像（内含 `+` 号），文本为 `关联其他账号`。点击后引导用户在主配置弹窗上方拉起“官方 API 授权”与“网页模拟登录捕获”的绑定通道。
*   **3. 官方 API 连接模式 (Official API Sync - Mode A)**：
    *   **使用场景**：商户拥有官方开发者凭证（如聚水潭 ERP、抖店资金账单、千川投放）。在选择“使用自己的账号”后，用户可点击“官方 API 授权”，直接输入申请好的 `AppKey` / `AppSecret`，或者点击跳转至对应三方开放平台进行 OAuth 2.0 授权一键绑定。
    *   **优势**：接口协议标准，Token 有效期长且支持后台自动刷新，运行最为稳定。
*   **4. 模拟登录与 Token 捕获模式 (Simulated Login & Token Capture - Mode B)**：
    *   **使用场景**：当官方未提供公开 API 接口，或接口权限申请门槛极高时（如罗盘核心指标、小红书蒲公英等）。用户点击“模拟登录连接”，系统在内嵌的官方登录窗口中引导用户扫码或登录，后端自动拦截并捕获该会话的 Cookie/Token。
    *   **🌟 导航菜单格式优先拉取与动态请求机制**：
        在模式 B 启动同步时，后台不会硬编码数据接口地址，而是**优先拉取并读取该平台工作台的最新导航菜单结构**，动态解析出目标报表对应的最新查询 Endpoint 和入参，再发起数据请求。这一机制极大地保证了系统的业务稳定性，最大程度降低了因平台控制台版面升级导致的同步中断风险。
*   **5. 敏感数据安全盾**：针对淘宝 `SessionKey` 或其他平台的密钥，后台进行 AES 加密存储，页面展示时采用掩码脱敏（如 `LTAI5t******`）。对于拦截捕获的 Cookie，前台完全隐藏，不提供任何明文显示或复制入口。

### (二) 核心数据源自动同步模块 (Automated Data Sync Engine)
负责对已开放 API 接口的数据源进行高稳定、全自动的数据搬运。
*   **1. 定时刷新配置 (Sync Scheduler)**：提供简化的自动刷新配置。用户可选择“每小时自动刷新”或“不自动刷新（仅手动触发）”，降低配置开销并规避多频次复杂定时冲突。
*   **2. 智能分页与限频规避 (Adaptive Fetcher)**：当单次拉取数据量超万条（如聚水潭发货报表）时，系统自动拆分分页请求，并根据目标平台 API 频控要求实施微秒级限速，防止账号因调用频次过高被第三方平台暂时拉黑。
*   **3. 增量去重落地 (Upsert Engine)**：系统写入飞书多维表格时，基于“联合唯一主键”（例如：订单号，或日期+账号名称）进行核对。若表格中已存在该记录则执行更新，不存在则执行插入，彻底杜绝数据重复。

### (三) 非API/受限平台数据桥接模块 (Restricted Data Ingestion Hub)
专门解决“业务急需数据，但平台官方未开放公共 API 接口（如抖音罗盘、小红书千帆核心指标）”这一行业痛点。
*   **1. 智能 Excel/CSV 拖放解析器**：运营人员从抖音罗盘或小红书千帆后台手工导出 Excel 后，直接拖入连接器页面。解析引擎自动读取表头，匹配字段映射规则，自动转换数据类型并同步至飞书，免去人工复制粘贴的工作。
*   **2. RPA 自动化搬运桥接**：系统预留标准 Webhook。当企业使用 RPA（机器人流程自动化）脚本在云端自动下载报表后，RPA 可将文件直接推送至连接器 API，实现数据的无人工干预接入。

### (四) 飞书多维表格映射配置模块 (Mapping & Connector Manager)
让非技术背景的运营人员也能自主设计数据流向。
*   **1. 可视化字段连线映射**：在配置弹窗中，左侧显示第三方平台接口字段（如 `settle_amount`），右侧显示飞书多维表格已有的列名（如 `结算金额`），用户通过简单的下拉框或拖拽连线即可建立映射。
*   **2. 动态时间过滤条件设置**：支持设置“动态时间偏移”，如“同步 T-1 报表（昨天的数据）”或“同步前天到今天的数据”，自动过滤无效的历史脏数据。

### (五) 异常自动归档与账户共享模块 (Error Archiving & Account Sharing)
为了精简前台操作与降低管理成本，本连接器的配置弹窗**完全不显示任何报警与预警配置项**。所有对于同步故障的管理均在后台与专用的异常跟踪表中闭环：

*   **1. 后端静默异常归档 (Automatic Error Logging to Bitable)**：
    *   在自动或手动同步过程中，若因 Cookie 过期、三方接口 500、网络超时或飞书写入频控触发限制而导致同步失败时，后端同步引擎会自动捕获异常。
    *   后端将自动提取故障元数据（包括：任务ID、异常时间、出错平台、账号店铺名称、错误大类、平台返回的原始 Response 文本等）。
    *   提取后，后端静默调用飞书多维表格 API，将该异常作为一行新增记录追加至企业指定的**【数据同步异常对账监控多维表格】**中。技术与运营人员可在该表格中统一排查、跟踪状态。如果需要通知，可直接在多维表格中通过自带的“自动化助手”配置“当新增报错行时自动通知相关负责人”。
*   **2. 极简账号内共享**：
    *   在绑定自己的全新账号时，用户勾选 `[ ] 允许该账号在企业内共享复用`。保存后，该账号便进入企业的共享账号数据库，供本企业内其他协作者免密直接挂载，无任何安全凭证暴露。

## 四、 26项具体数据对接功能矩阵 (含API与操作说明)

系统将多维表格中提供的 26 项具体平台页面/模块，根据数据源形态与接口可用性，统一收敛为 **3 种标准功能方案** 进行实施：

### 1. 资金/账户账单通用方案
> **适用范围**：抖店余额、待结算资金、京麦销售明细、千牛支付宝/微信账单、小红书余额与待结算（共计 **7 项**）。

*   **同步形态**：全自动 API 同步（支付宝/微信/抖店/京麦）/ RPA 辅助（小红书资金）。
*   **用户操作流程**：
    1.  业务人员在配置弹窗中选择并绑定商户授权。
    2.  在飞书多维表格中新建一张【资金对账表】。
    3.  在配置弹窗中将接口标准字段（变动金额、待结算金额、对账日期、流水单号）映射至表格。
    4.  系统每日自动同步并增量更新。

### 2. 推广投放报表通用方案
> **适用范围**：阿里妈妈万象台、巨量千川（素材、全域、单品）、小红书乘风与聚光（分日、月汇总、计划/账户报表）（共计 **12 项**）。

*   **同步形态**：全自动 API 同步。
*   **用户操作流程**：
    1.  在连接器配置弹窗中绑定巨量引擎/小红书营销账号。
    2.  系统按设定的定时规则，自动调取对应平台的 Marketing API 报表接口。
    3.  自动将千川/聚光/万象台的消耗、展现、点击、转化金额及 ROI 写入多维表格。
    4.  对于历史数据变动（如广告归因周期导致前几日 GMV 发生变化），系统自动在每日凌晨回溯同步前 3 天的数据进行覆盖更新。

### 3. ERP报表与受限/非公开模块通用方案
> **适用范围**：聚水潭ERP（发货、出库、退货）、电商罗盘（交易概览、商品核心）、小红书千帆经营数据、小红书蒲公英（共计 **9 项**）。

*   **同步形态**：
    *   **ERP 模块**：官方 API 实时拉取。
    *   **受限看板/非公开模块**：**优先采用“模拟登录 Token 捕获”自动同步**；若用户不愿提供登录凭证，可降级采用“RPA 自动化”或“Excel 手工导入”。
*   **用户操作流程**：
    *   **聚水潭 ERP**：完成开放平台授权后，设定每 15 分钟增量拉取一次销售出库单与退货单，多维表格库存状态实时自动刷新。
    *   **电商罗盘 / 小红书千帆与蒲公英 (核心设计)**：
        *   **自动化模式 (推荐)**：
            1. 用户在多维表格同步页面选择对应的平台（如抖音罗盘、小红书蒲公英）。
            2. 点击“添加登录连接”，在内嵌的官方安全登录框中输入账号密码或扫码登录。
            3. 连接器自动拦截捕获后台会话 Token/Cookie，并配置同步的具体参数（如同步频率、目标多维表格字段）。
            4. 后台服务将在后台定期使用该 Token 模拟请求，将接口原本不存在的详细数据自动灌入表格。
        *   **手动导入模式 (降级备用)**：
            1. 运营在配置弹窗中打开“非API导入助手”，将从罗盘或蒲公英手动导出的原始 Excel/CSV 拖入上传区。
            2. 系统解析器根据字段模板快速解析并覆盖写入表格。

## 五、 数据安全与运行保障

1. **多租户数据隔离**：
   各商家的授权 Token 与多维表格数据在数据库中拥有严格的租户标识隔离，禁止跨企业访问。
2. **多维表格写入高可用保障 (Queueing)**：
   飞书限制了每分钟的 API 请求频率。连接器内部设计了**消息写入队列**。即使遇到促销大促期间产生巨量单据写入，系统也会将写入操作置于队列中平滑下发，防止触发飞书 `Spam` 保护或写入丢失。
3. **操作日志追溯**：
   系统会详细记录业务人员的每一次手工同步触发、管理员的映射修改及文件导入历史，确保资金账单和投放报表的数据改动有迹可循。
